# 更新日志

本项目从 `1.0.0` 开始记录版本变更。每次发布前都要在这里新增对应版本的条目，并同步更新 README 中列出的版本号位置。

## 1.1.0 - 2026-06-13

- 从用户录制的 21 音卡林巴录音中切分真实采样，按页面琴键从左到右的音名保存为 `d6.mp3` 到 `e6.mp3`。
- 示范播放和琴键点击优先使用真实采样音色，单个采样加载失败时仅该音回退到 Web Audio 合成音。
- 新增 `scripts/split-samples.mjs`，用于重新检测拨弦瞬态并生成 `assets/samples/` 下的 MP3 采样。
- 将 21 个采样加入 Service Worker 离线缓存，并将缓存版本更新为 `kalimba-tone-trace-v1.1.0`。

## 1.0.5 - 2026-06-13

- 添加歌曲保存时新增短暂的 `AI thinking......` 加载弹窗，让导入 JSON 的反馈更明确。
- 导入期间会暂时禁用保存按钮和 JSON 输入框，避免重复点击。
- 添加本地曲库管理，可删除本机导入歌曲，系统内置歌曲不可删除。
- 将 Service Worker 离线缓存版本更新为 `kalimba-tone-trace-v1.0.5`。

## 1.0.4 - 2026-06-13

- 新增曲库难度分级，内置和本地导入歌曲都会带有 `easy`、`medium`、`hard` 难度。
- 曲库页新增“全部 / 简单 / 中等 / 困难”分组，搜索结果会先全局过滤，再按难度分组展示。
- 添加歌曲提示词和 JSON 导入支持 `difficulty` 字段，缺省时会根据速度、密度、跨度和跳音估算难度。
- 从曲库选择歌曲进入练习页时，手机竖屏下会直接启用软件横屏练习模式，省去二次点击横屏提示。
- 将 Service Worker 离线缓存版本更新为 `kalimba-tone-trace-v1.0.4`。

## 1.0.3 - 2026-06-13

- 新增本地歌曲导入能力，曲库页可复制 AI 提示词并粘贴标准 JSON 保存到本地曲库。
- 新增 `src/song-store.js`，统一管理内置曲库、本地曲库、JSON 校验和简谱 `notation` 到练习 `steps` 的编译。
- 导入 JSON 仅通过 `JSON.parse` 解析，并校验 C 调、音域、节拍和必填字段。
- 练习页和曲库页改为读取“内置歌曲 + 本地歌曲”的合并曲库。
- 将 Service Worker 离线缓存版本更新为 `kalimba-tone-trace-v1.0.3`。

## 1.0.2 - 2026-06-13

- 新增独立曲库页面 `songs.html`，支持搜索当前内置歌曲并选择进入练习页。
- 练习页移除原生歌曲下拉框，改为显示当前曲目并通过“换曲”进入曲库。
- 曲库数据增加上传者字段，当前内置歌曲上传者为 `system`。
- 优化导航缓存逻辑，离线访问曲库页时优先回退到对应页面。
- 将 Service Worker 离线缓存版本更新为 `kalimba-tone-trace-v1.0.2`。

## 1.0.1 - 2026-06-13

- 修复鸿蒙手机/PWA 中点击“进入横屏”后可能卡在竖屏提示层的问题。
- 横屏按钮现在会优先请求系统全屏和原生横屏锁定。
- 当浏览器拒绝原生横屏锁定时，启用软件横屏模式，把练习界面旋转成可用的横屏画布。
- 增加 PWA 新版本提示，发现新版本时可点击“立即更新”切换到新版缓存。
- 将 Service Worker 离线缓存版本更新为 `kalimba-tone-trace-v1.0.1`。

## 1.0.0 - 2026-06-13

- 建立 `Kalimba ToneTrace / 卡林巴循音` 工程结构。
- 将原单文件网页拆分为 `index.html`、`src/app.js`、`src/songs.js`、`src/pitch.js` 和 `src/styles.css`。
- 新增 PWA 配置、离线缓存、手机桌面图标和旧地址跳转页。
- 新增本地自绘木质卡林巴图标。
- 保留卡林巴跟练、示范播放、麦克风识别、横屏练习和琴键宽度缩放等核心功能。
